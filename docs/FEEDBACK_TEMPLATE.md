## 📝 1. 일반 피드백 양식

**제출 방법:** 텔레그램 `#자유-피드백` 또는 [구글폼 링크]

```markdown
### 일반 피드백

**이름:** [이름 또는 닉네임]
**제출 날짜:** YYYY-MM-DD

#### 1. 이번 주 검시AI를 얼마나 사용했나요?
- [ ] 거의 매일 (5일 이상)
- [ ] 자주 (3-4일)
- [ ] 가끔 (1-2일)
- [ ] 거의 안 씀

#### 2. 주로 어떤 과목을 학습했나요? (복수 선택 가능)
- [ ] 해부학
- [ ] 생리학
- [ ] 병리학
- [ ] 예방의학
- [ ] 의료법규
- [ ] 공중보건학

#### 3. 가장 유용했던 기능은?
[자유 입력]

예시: "해부학 AI 튜터가 구조 설명을 단계별로 해줘서 이해하기 쉬웠어요."

#### 4. 가장 불편했던 점은?
[자유 입력]

예시: "모바일에서 이미지가 너무 작아서 보기 어려웠어요."

#### 5. 만족도 평가 (1-5점)

| 항목 | 점수 (1=매우 불만, 5=매우 만족) |
|------|--------------------------------|
| AI 튜터 응답 품질 | __점 |
| UI/UX (사용 편의성) | __점 |
| 응답 속도 | __점 |
| 전반적 만족도 | __점 |

#### 6. 추가하고 싶은 기능이 있다면?
[자유 입력]

#### 7. 친구에게 검시AI를 추천할 의향이 있나요? (0-10점)
0 (절대 안함) ━━━━━━━━━━ 10 (강력 추천)

__점

#### 8. 기타 의견
[자유 입력]
```

## 🐛 2. 버그 리포트 템플릿

**제출 방법:** 텔레그램 `#버그-리포트` 또는 [GitHub Issues]

```markdown
### 🐛 버그 리포트

**제목:** [간단한 버그 설명]
예: "AI 튜터 응답 중 특수문자 깨짐"

---

#### 1. 버그 설명
[무엇이 잘못되었는지 명확히 설명]

예시:
"AI 튜터에게 '△의 의미는?' 질문했더니 응답에서 △가 □로 표시됨"

#### 2. 재현 단계
[버그를 다시 발생시키는 방법]

예시:
1. 해부학 AI 튜터 진입
2. "△의 의미는?" 입력
3. AI 응답 확인

#### 3. 예상 동작
[원래 어떻게 작동해야 하는지]

예: "△가 정상적으로 표시되어야 함"

#### 4. 실제 동작
[실제로 어떻게 작동했는지]

예: "△ 대신 □로 표시됨"

#### 5. 스크린샷/동영상
[첨부 또는 링크]

#### 6. 환경 정보
- **브라우저:** [Chrome 120 / Safari 17 / Firefox 121 등]
- **OS:** [macOS 14.2 / Windows 11 / iOS 17 / Android 13]
- **디바이스:** [MacBook Pro M1 / iPhone 14 / Galaxy S23]
- **화면 크기:** [데스크톱 / 태블릿 / 모바일]

#### 7. 재현 빈도
- [ ] 항상 발생
- [ ] 자주 발생 (50% 이상)
- [ ] 가끔 발생
- [ ] 한 번만 발생

#### 8. 심각도 (본인 판단)
- [ ] Critical - 앱 사용 불가
- [ ] High - 주요 기능 장애
- [ ] Medium - 일부 불편
- [ ] Low - 사소한 문제

#### 9. 추가 정보
[기타 참고 사항]
```

**예시 버그 리포트:**

```markdown
### 🐛 버그 리포트

**제목:** 모바일에서 로그인 후 화면 멈춤

---

#### 1. 버그 설명
모바일(iPhone 14)에서 로그인 성공 후 대시보드로 이동하지 않고 화면이 멈춤

#### 2. 재현 단계
1. 모바일 Safari로 beta.gumsi-ai.com 접속
2. 이메일/비밀번호 입력 후 "로그인" 탭
3. 로딩 스피너 계속 돌고 화면 전환 안 됨

#### 3. 예상 동작
로그인 후 대시보드로 자동 이동

#### 4. 실제 동작
로딩 스피너만 계속 돌고 3분 기다려도 변화 없음

#### 5. 스크린샷
[스크린샷 첨부]

#### 6. 환경 정보
- 브라우저: Safari 17.2
- OS: iOS 17.3
- 디바이스: iPhone 14
- 화면 크기: 모바일 (390x844)

#### 7. 재현 빈도
- [x] 항상 발생

#### 8. 심각도
- [x] High - 주요 기능 장애

#### 9. 추가 정보
- 데스크톱 Chrome에서는 정상 작동함
- 모바일 Chrome(Android)에서는 테스트 안 해봄
```

## 💡 3. 기능 요청 템플릿

**제출 방법:** 텔레그램 `#기능-제안` 또는 [구글폼 링크]

```markdown
### 💡 기능 요청

**제목:** [요청하는 기능 한 줄 요약]
예: "학습 목표 설정 및 진도 알림 기능"

---

#### 1. 기능 설명
[어떤 기능을 원하는지 자세히]

예시:
"시험 날짜를 설정하고, 일일/주간 학습 목표를 세울 수 있는 기능.
목표 달성률이 낮으면 알림을 보내주면 좋겠습니다."

#### 2. 사용 시나리오
[언제, 어떻게 사용할지]

예시:
"저는 3월 15일 검시 시험 예정이에요.
앱에 시험 날짜 입력 → 자동으로 '하루 2시간 학습 권장' 제안
→ 목표 미달성 시 저녁 9시 알림"

#### 3. 기대 효과
[이 기능이 있으면 무엇이 좋아지는지]

예시:
"학습 동기 부여 + 일정 관리가 쉬워질 것 같아요."

#### 4. 우선순위 (본인 판단)
- [ ] 필수 (없으면 앱 못 씀)
- [ ] 중요 (있으면 훨씬 좋음)
- [ ] 부가 (있으면 좋지만 없어도 됨)

#### 5. 참고 사례
[비슷한 기능이 있는 다른 앱/서비스]

예: "Duolingo의 연속 학습일 기록 같은 느낌"

#### 6. 추가 의견
[기타 참고 사항]
```

**예시 기능 요청:**

```markdown
### 💡 기능 요청

**제목:** 오답 노트 복습 알림

---

#### 1. 기능 설명
틀린 문제를 자동으로 "오답 노트"에 저장하고,
며칠 후 다시 풀어보라는 알림을 보내주는 기능

#### 2. 사용 시나리오
1. 기출문제 풀다가 틀린 문제 → 자동으로 오답 노트에 추가
2. 3일 후 "오답 노트 복습하기" 알림 도착
3. 앱 열면 틀렸던 문제들 다시 풀기

#### 3. 기대 효과
반복 학습으로 약한 부분 보완 가능.
Anki처럼 간격 반복 학습 효과 기대.

#### 4. 우선순위
- [x] 중요 (있으면 훨씬 좋음)

#### 5. 참고 사례
Anki (플래시카드 앱)의 spaced repetition 알고리즘

#### 6. 추가 의견
복습 간격을 사용자가 설정할 수 있으면 더 좋을 것 같아요 (1일, 3일, 7일 등)
```

## 🗣️ 4. 인터뷰 질문지 (1:1 심층 인터뷰용)

   - 장소: 집/도서관/카페/이동 중
   - 디바이스: 모바일/태블릿/노트북

5. 6개 과목 중 어느 과목을 가장 많이 사용했나요? 이유는?

6. AI 튜터와의 대화에서 가장 만족스러웠던 순간은?
   - 구체적인 예시를 들어주세요

7. 반대로, 가장 실망스러웠던 순간은?
   - 어떤 질문에 어떤 답변이 왔나요?

8. 기출문제 풀이 기능은 어떠셨나요?
   - 해설이 도움이 되었나요?

9. UI/UX는 어떠셨나요?
   - 사용하기 편했나요? 불편했던 점은?

11. 검시AI가 기존 학습 방법과 비교해서 어떤 점이 다른가요?
    - 더 나은 점 / 부족한 점

12. 검시AI에 꼭 추가되었으면 하는 기능이 있다면?

#### 가격 및 전환 의향 (3분)
13. 검시AI가 유료화된다면 얼마까지 지불할 의향이 있으신가요?
    - 월 구독: ______원
    - 일시 구매: ______원

14. 정식 출시 후에도 계속 사용하실 건가요?
