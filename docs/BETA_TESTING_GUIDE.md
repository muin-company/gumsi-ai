## 베타 테스터 모집 후 해야 할 일

### 1. 테스터 데이터베이스 구축 (D+0)
- [ ] 스프레드시트 생성 (이름, 이메일, 전화번호, 학습 과목, 시험 예정일)
- [ ] 테스터 그룹 분류
  - **Group A**: 의료법규, 공중보건학 (행정 과목)
  - **Group B**: 해부학, 생리학, 병리학, 예방의학 (이론 과목)
  - **All**: 전체 6과목
- [ ] 텔레그램/디스코드 테스터 채널 개설
- [ ] 테스터 전용 Notion/Slack 워크스페이스 초대

### 2. 접근 권한 설정 (D+1)
- [ ] 베타 환경 URL 공유 (예: `beta.gumsi-ai.com`)
- [ ] 테스터별 계정 생성 또는 회원가입 링크 전달
- [ ] 베타 플래그 활성화 (feature flag)
- [ ] 테스터 전용 프로모 코드 발급 (무료 이용)

### 3. 킥오프 미팅 준비 (D+2~3)
- [ ] 온라인 미팅 일정 조율 (30분~1시간)
- [ ] 발표 자료 준비
  - 검시AI 소개 (5분)
  - 베타 테스트 목적 및 기간 (5분)
  - 사용 방법 시연 (10분)
  - 피드백 방법 안내 (5분)
  - Q&A (10분)
- [ ] 녹화 준비 (참석 못한 테스터용)

### 4. 리소스 배포 (D+3)
- [ ] 온보딩 가이드 PDF/Notion 공유
- [ ] 피드백 양식 링크 공유
- [ ] 버그 리포트 템플릿 공유
- [ ] FAQ 문서 공유

## 테스터 온보딩 프로세스

### Step 1: 환영 메시지 발송 (자동화)
```
안녕하세요 [이름]님!

검시AI 베타 테스트에 참여해주셔서 감사합니다 🎉

[베타 환경 URL]
아이디: [이메일]
임시 비밀번호: [자동생성]

시작 가이드: [Notion 링크]
피드백 채널: [텔레그램/디스코드]

궁금한 점은 언제든 [연락처]로 문의해주세요!
```

### Step 2: 온보딩 가이드 제공
**포함 내용:**
1. 검시AI란? (1분 소개 영상 또는 GIF)
2. 회원가입/로그인 방법
3. 과목 선택 및 AI 튜터 시작하기
4. 기출문제 풀이 방법
5. 진도 확인 및 학습 통계 보기
6. 피드백 제출 방법

### Step 3: 퀵스타트 미션 (첫 3일)
테스터가 주요 기능을 모두 경험하도록 유도:
- [ ] 회원가입 완료
- [ ] AI 튜터와 첫 대화 (아무 과목)
- [ ] 기출문제 1문제 풀기
- [ ] 피드백 1개 제출

**완료 시 인센티브:** 스타벅스 기프티콘 또는 베타 테스터 굿즈

### Step 4: 체크인 미팅 (D+7)
- 1:1 또는 소그룹 미팅 (15분)
- 첫 인상 묻기
- 막힌 부분 해결
- 다음 주 테스트 목표 설정

## 피드백 수집 방법

### 1. 정량적 피드백 (설문)

**주간 설문 (매주 금요일 발송)**
- 이번 주 사용 빈도: □ 매일 □ 3-4회 □ 1-2회 □ 안 씀
- 만족도 (1-5점): AI 응답 품질 / UI/UX / 속도 / 전반적 만족도
- 가장 유용했던 기능: __________
- 가장 불편했던 점: __________
- NPS: 0-10점 (친구에게 추천할 의향)

**도구:** Google Forms, Typeform, Tally

### 2. 정성적 피드백 (인터뷰)

**격주 1:1 인터뷰 (30분)**
- 테스터당 최소 2회 (베타 초반 + 후반)
- 질문 예시:
  - "어떤 상황에서 검시AI를 사용하나요?"
  - "AI 튜터 응답 중 가장 마음에 들었던/들지 않았던 답변은?"
  - "기존 학습 방법 대비 어떤 점이 다른가요?"
  - "유료 전환 시 얼마까지 지불할 의향이 있나요?"

**녹음 동의 필수 + 전사 후 Notion에 정리**

### 3. 실시간 피드백 (채널)

**텔레그램/디스코드 채널 운영**
- `#버그-리포트`: 버그 신고 전용
- `#기능-제안`: 기능 요청
- `#자유-피드백`: 자유로운 의견
- `#공지`: 업데이트 공지

**응답 시간 목표:** 24시간 이내

### 4. 행동 데이터 분석

**주간 분석 지표:**
- DAU/WAU (일일/주간 활성 사용자)
- 과목별 사용 비율
- AI 튜터 대화 평균 턴 수
- 기출문제 풀이 완료율
- 이탈 지점 (어느 페이지에서 나가는지)

**도구:** Google Analytics, Mixpanel, PostHog

## 버그 리포팅 프로세스

### 1. 버그 발견 → 신고
테스터가 `FEEDBACK_TEMPLATE.md`의 버그 리포트 양식 작성  
→ 텔레그램 `#버그-리포트` 채널 또는 구글폼 제출

### 2. 버그 확인 (24시간 내)
- [ ] 재현 가능 여부 확인
- [ ] 심각도 분류 (Critical / High / Medium / Low)
- [ ] 담당자 할당

### 3. 우선순위 결정
아래 "우선순위 결정 기준" 참고

### 4. 수정 및 배포
- [ ] 수정 완료
- [ ] QA 테스트
- [ ] 베타 환경 배포
- [ ] 테스터에게 "수정 완료" 공지

### 5. 테스터 피드백
- [ ] 수정 확인 요청
- [ ] 추가 이슈 없으면 Close

**버그 트래킹 도구:** GitHub Issues, Linear, Jira

## 우선순위 결정 기준

### Severity (심각도)
| Level | 정의 | 예시 | 대응 시간 |
|-------|------|------|----------|
| **P0 - Critical** | 서비스 사용 불가 | 로그인 불가, 서버 다운, 결제 오류 | 즉시 (4시간 내) |
| **P1 - High** | 핵심 기능 장애 | AI 응답 안 옴, 문제 풀이 불가 | 24시간 내 |
| **P2 - Medium** | 일부 기능 장애 | 진도 저장 안 됨, 이미지 안 뜸 | 3일 내 |
| **P3 - Low** | UI 버그, 불편 | 버튼 색상 이상, 오타, 레이아웃 깨짐 | 다음 스프린트 |

### Impact (영향 범위)
- **전체 사용자**: +2 우선순위
- **특정 과목/기능**: +1 우선순위
- **극소수 엣지 케이스**: +0

### Effort (수정 난이도)
- **Quick Fix (1시간 내)**: 바로 처리
- **Medium (1일)**: 스프린트에 포함
- **Complex (3일+)**: 백로그 등록 후 PM과 협의

### 최종 우선순위 = Severity + Impact - Effort

## 주간 체크리스트

### 월요일: 주간 계획
- [ ] 지난주 피드백 리뷰 (팀 미팅 30분)
- [ ] 이번 주 수정할 버그/기능 선정
- [ ] 테스터에게 이번 주 테스트 포커스 안내
  - 예: "이번 주는 해부학 AI 튜터 집중 테스트!"

### 화요일: 데이터 분석
- [ ] GA/Mixpanel 주간 리포트 확인
- [ ] 이탈률 높은 페이지 파악
- [ ] 사용 빈도 낮은 테스터 1:1 연락

### 수요일: 중간 체크인
- [ ] 텔레그램 채널 모니터링
- [ ] 미응답 버그 리포트 확인
- [ ] 긴급 이슈 있으면 핫픽스

### 목요일: 업데이트 준비
- [ ] 주간 배포 준비 (버그 수정, 기능 개선)
- [ ] 릴리즈 노트 작성
- [ ] 베타 환경 배포

### 금요일: 피드백 수집
- [ ] 주간 설문 발송 (오전 10시)
- [ ] 릴리즈 노트 공유
- [ ] 테스터 감사 메시지 발송
- [ ] 다음주 테스트 목표 예고

### 주말: 데이터 정리
- [ ] 이번 주 버그/기능 요청 스프레드시트 업데이트
- [ ] 우선순위 재조정
- [ ] 월요일 미팅 자료 준비

## 베타 테스트 성공 지표 (KPI)

### 참여도
- **목표:** 주간 활성 사용자 80% 이상
- **측정:** 주 1회 이상 로그인한 테스터 비율

### 만족도
- **목표:** NPS 50+ (추천 의향 높음)
- **측정:** 주간 설문 NPS 점수

### 피드백 품질
- **목표:** 주간 평균 버그 리포트 10건 이상
- **목표:** 주간 평균 기능 제안 5건 이상

### 제품 개선
- **목표:** 주간 5개 이상 버그 수정
- **목표:** 베타 기간 내 10개 이상 기능 개선
