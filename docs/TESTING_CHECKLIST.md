# 검시AI 테스트 체크리스트

베타 테스트 및 정기 QA용 종합 체크리스트. 각 항목을 테스트하며 `[ ]`를 `[x]`로 체크.

---

## 📋 기능 테스트

### 1. 회원가입/로그인

#### 회원가입
- [ ] 이메일 회원가입 성공
- [ ] 비밀번호 유효성 검사 (최소 8자, 영문+숫자)
- [ ] 중복 이메일 가입 차단
- [ ] 이메일 인증 발송 (해당시)
- [ ] 이메일 인증 링크 클릭 시 활성화
- [ ] 소셜 로그인 (구글) - 해당시
- [ ] 소셜 로그인 (카카오) - 해당시

#### 로그인
- [ ] 정상 로그인 성공
- [ ] 잘못된 비밀번호 시 에러 메시지
- [ ] 존재하지 않는 이메일 시 에러 메시지
- [ ] "비밀번호 찾기" 기능 작동
- [ ] 비밀번호 재설정 이메일 발송
- [ ] 비밀번호 재설정 완료 후 로그인 가능

#### 세션 관리
- [ ] 로그인 상태 유지 (새로고침 후)
- [ ] 로그아웃 기능 작동
- [ ] 로그아웃 후 보호된 페이지 접근 차단
- [ ] 세션 만료 시 자동 로그아웃 (해당시)

- [ ] 멀티턴 대화 가능 (문맥 유지)

- [ ] AI 응답 오류 시 재시도 버튼
- [ ] 네트워크 끊김 시 에러 처리

## ⚡ 성능 테스트

### 페이지 로딩 속도

**목표: LCP < 2.5초 (Lighthouse 기준)**

- [ ] 홈페이지 로딩 < 1.5초
- [ ] 로그인 페이지 < 1초
- [ ] AI 튜터 채팅 < 2초
- [ ] 기출문제 목록 < 2초
- [ ] 대시보드 < 2.5초

**도구:** Lighthouse, PageSpeed Insights, WebPageTest

#### 최적화 체크
- [ ] 이미지 lazy loading
- [ ] 번들 크기 < 500KB (gzipped)
- [ ] CDN 사용 (정적 자산)
- [ ] 서버 사이드 렌더링 (SSR) 또는 정적 생성 (SSG)
- [ ] HTTP/2 또는 HTTP/3

---

### AI 응답 속도

**목표: 평균 응답 시간 < 3초**

- [ ] 첫 토큰 수신 (TTFT) < 1초
- [ ] 전체 응답 완료 < 3초
- [ ] 스트리밍 응답 끊김 없음
- [ ] 동시 요청 시 응답 시간 유지

**부하 테스트 시나리오:**
- [ ] 10명 동시 사용자
- [ ] 50명 동시 사용자
- [ ] 100명 동시 사용자

**도구:** Locust, k6, Artillery

---

### 동시 사용자 테스트

**목표: 100명 동시 접속 시 응답 시간 증가 < 20%**

#### 시나리오 1: 일반 사용
- [ ] 50% 로그인 + 브라우징
- [ ] 30% AI 튜터 채팅
- [ ] 20% 문제 풀이

#### 시나리오 2: 피크 타임
- [ ] 100명 동시 AI 채팅
- [ ] 서버 CPU/메모리 사용률 < 80%
- [ ] 에러율 < 1%

#### 모니터링
- [ ] 응답 시간 p50, p95, p99
- [ ] 에러 로그 수집
- [ ] DB 쿼리 성능

**도구:** New Relic, DataDog, Sentry

## 🔒 보안 테스트

### 인증/인가

- [ ] 비로그인 시 API 호출 차단
- [ ] JWT 토큰 검증
- [ ] 토큰 만료 처리
- [ ] Refresh token 갱신
- [ ] 타 사용자 데이터 접근 차단
- [ ] 관리자 권한 분리 (해당시)

---

### XSS (Cross-Site Scripting) 방어

- [ ] 사용자 입력 HTML 이스케이핑
- [ ] `<script>alert('XSS')</script>` 입력 시 실행 안 됨
- [ ] Content-Security-Policy 헤더 설정
- [ ] 마크다운 렌더링 시 sanitize

**테스트 페이로드:**
```html
<img src=x onerror=alert('XSS')>
<svg/onload=alert('XSS')>
```

---

### CSRF (Cross-Site Request Forgery) 방어

- [ ] CSRF 토큰 생성 및 검증
- [ ] SameSite 쿠키 설정
- [ ] 중요 작업 시 재인증 요구 (해당시)

---

### API Rate Limiting

- [ ] 로그인 시도 제한 (5회/분)
- [ ] AI 채팅 요청 제한 (10회/분)
- [ ] 회원가입 제한 (3회/시간/IP)
- [ ] Rate limit 초과 시 429 응답

**도구:** Redis + express-rate-limit

---

### 기타 보안

- [ ] HTTPS 강제 (HTTP → HTTPS 리다이렉트)
- [ ] 비밀번호 해싱 (bcrypt, argon2)
- [ ] 민감 정보 로그 제외
- [ ] 환경 변수 안전 관리 (.env 노출 방지)
- [ ] SQL Injection 방어 (Prepared statements)
- [ ] CORS 설정 (허용된 도메인만)
